3 approches pour restructurer le dashboard

Partant de : 4 tabs (Overview, Analyses, Interviews, Applications) → quel nouveau shape ?

A
3 tabs — Home · Analyses · Applications Recommandé
Merge Interviews dans Analyses, Overview devient Home intelligente
🏠 Home 📊 Analyses (+ interviews) 📋 Applications 🎤 Interviews
Home = command center : candidatures urgentes, score en évolution, raccourcis. Analyses = historique + évolution + interviews imbriqués dans le détail de chaque analyse. Applications = CRM tel quel mais optimisé.
✓ Pour
  • Nav épurée, 3 tabs clairs
  • Interviews contextualisés
  • Home = valeur immédiate au retour
  • Moins de redondance
✗ Contre
  • Home à concevoir soigneusement
  • Migration UX pour users existants
B
2 tabs — Analyses · Applications
Supprimer l'Overview, rendre chaque tab auto-suffisant
📊 Analyses (+ stats + interviews) 📋 Applications (+ stats CRM) 🏠 Overview 🎤 Interviews
Chaque tab a son propre header avec ses métriques clés. Analyses affiche en haut : score moyen, trend, radar. Applications affiche : nb actives, taux de réponse, funnel.
✓ Pour
  • Ultra minimaliste
  • Pas de page "vide" à gérer
✗ Contre
  • Pas de "landing" au retour
  • Perd la vision cross-tab
  • Moins d'engagement
C
4 tabs redesignés — garder la structure, tout améliorer
Évolution douce : Overview amélioré, Interviews enrichi, même nav
🏠 Overview (amélioré) 📊 Analyses (amélioré) 🎤 Interviews (enrichi) 📋 Applications
Garder 4 tabs mais rendre chacun meilleur : Overview devient plus actionnable, Interviews montre le contexte job, Analyses lie mieux les deux. Moins de refactoring, plus de polish.
✓ Pour
  • Moins de disruption
  • Plus rapide à shipper
  • Pas de migration UX
✗ Contre
  • Redondance Overview / Analyses reste
  • Interviews toujours orphelin
  • 4 tabs c'est une tab de trop
